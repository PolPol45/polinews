# Calendar - 4 Week Pilot

## Cadence
- Day 1 kickoff
- Checkpoint every 2 days
- Weekly checkpoint on days 7, 14, 21, 28

## Weekly Program (v0.2 alignment)
- Week 1: ingestion foundation (`W1-01..W1-12`) including collector and raw DB ingest.
- Week 2: story plus quiz plus ledger core and data-layer schema readiness (`W2-01..W2-17`).
- Week 3: antifraud plus analytics data capture plus annotation task pipeline (`W3-01..W3-16`).
- Week 4: pilot run plus decision gates plus data monetization and phase2 readiness (`W4-01..W4-15`).
- Month 2+: annotation buyer integration, dataset licensing, and on-chain voucher rollout prep.

## Weekly Deliverables (Founder + Codex)
| Week | Scope | Main owner | Deliverable to close week |
|---|---|---|---|
| 1 | RSS ingestion reliability and legal/source guardrails | Codex + Founder | Active feed runs stable, raw ingest evidence, source and attribution policy frozen |
| 2 | Story UX + quiz + ledger + schema/api readiness for data monetization | Codex + Founder | End-to-end quiz->credits flow working, schema and API extended for analytics/voucher |
| 3 | Fraud controls + analytics capture + annotation data structures | Codex + Founder | Fraud queue operational, data capture tables populated, first annotation pipeline checks |
| 4 | Pilot execution + KPI gates + compliance and phase2 decision package | Founder + Codex | Decision memo with usability/sustainability/risk + data-revenue view + phase2 readiness |

## Revenue-Model Alignment Checks (every checkpoint)
1. Publisher subscription loop still works with no wallet/KYC in MVP.
2. Dataset and analytics outputs remain pseudonymized and export-safe.
3. Annotation tasks do not break quiz UX (max one task-style question in flow).
4. Phase2 voucher/on-chain design stays isolated from MVP runtime path.

## Fixed slots (local time)
- 09:00 ingestion check
- 11:00 KPI drift check
- 14:00 fraud review
- 17:00 budget/redeem check
- 19:00 daily report

## Checkpoint agenda (30 min)
1. Completed tasks vs planned tasks
2. Blockers and owner actions
3. KPI trend and anomalies
4. Risk review
5. Next 48h priorities

## Mandatory outputs per checkpoint
- Updated task status
- Updated blocker list
- Evidence links collected
- Next checkpoint goals
