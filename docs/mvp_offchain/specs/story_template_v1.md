## 1. Story page structure — fixed section order

Sections must render in this exact order. No reordering allowed without a new version.

| # | Section | Source |
|---|---|---|
| 1 | **Topic badge** | `topic_slug` → fixed color map (see below) |
| 2 | **Headline** | `stories.headline` |
| 3 | **Snippet** | `stories.summary` (≤320 chars, hard cut) |
| 4 | **Key points** | Generated by `gpt-4o-mini` at story creation time (3–5 bullets) |
| 5 | **Attribution block** | Label + source links (see attribution_policy_v1 v2) |
| 6 | **Verification box** | Quiz or comment CTA |
| 7 | **Credits teaser** | Conditional — see auth rules below |

---

## 2. Section specs

### Topic badge
- One badge per story, top-left of headline.
- Text: `topic_slug` in human-readable form (e.g. `politics` → `Politics`).
- Fixed color per topic — do not use dynamic or random colors.

| Slug | Badge color (hex) |
|---|---|
| `politics` | `#534AB7` (purple) |
| `economy` | `#185FA5` (blue) |
| `world` | `#0F6E56` (teal) |
| `technology` | `#BA7517` (amber) |
| `health` | `#A32D2D` (red) |
| `climate` | `#3B6D11` (green) |
| `markets` | `#993C1D` (coral) |
| `security` | `#444441` (gray) |
| `business` | `#72243E` (pink) |
| `science` | `#085041` (dark teal) |

### Headline
- Rendered as `<h1>`.
- Required. If empty: story is `not_publishable`.

### Snippet
- Max 320 chars — hard cut at character level, no word-wrap.
- Required. If empty: story is `not_publishable`.

### Key points
- Generated by `gpt-4o-mini` at story creation time using headline + snippet as input.
- 3 minimum, 5 maximum. Target: 3–5.
- Rendered as an unordered list.
- Each bullet must not duplicate another bullet in the same story.
- No claims without a source anchor in the attribution block.
- Do not insert unverifiable facts.
- If generation fails or returns < 3 points: story is `not_publishable`.

### Attribution block
- Exact label text: `Summary generated by Poli-News` — no variations.
- Source links rendered immediately below the label, inline or as a list.
- Format per link: `[Source name] → [outbound URL]` (opens in new tab).
- At least 1 source link required. If missing: story is `not_publishable`.
- Block must appear **above** the verification box — always.

### Verification box
- Contains: quiz CTA or comment CTA (determined by quiz availability flag).
- Rendered below attribution block.

### Credits teaser
- **If user is authenticated:** show current credit balance + hint text (e.g. `"You have 30 credits — answer to earn more"`).
- **If user is not authenticated:** show login invite (e.g. `"Sign in to earn credits for reading"`). Do not show balance.
- Rendered below verification box.

## 3. Publishable criteria — minimum viable story

A story can only be served if ALL of the following are true:

```
headline        IS NOT NULL AND IS NOT EMPTY
snippet         IS NOT NULL AND IS NOT EMPTY
source_links    COUNT >= 1
key_points      COUNT >= 3
```

If any condition fails: set `stories.status = 'not_publishable'` and do not serve the story.
Log reason: `NOT_PUBLISHABLE story_id=<id> reason=<missing_field>`.

---

## 4. Metadata fields

| Field | Type | Notes |
|---|---|---|
| `story_id` | TEXT | Primary key |
| `topic_slug` | TEXT | One of the 10 frozen slugs |
| `source_count` | INTEGER | Number of source links |
| `published_at` | TIMESTAMP | Source timestamp or ingestion fallback |
| `generated_at` | TIMESTAMP | Time of key-point generation |
| `status` | TEXT | `publishable` or `not_publishable` |
