# kpi_spec_v1

## Primary KPIs

| KPI | Formula | Target | Alert |
|---|---|---|---|
| Attempt Rate | `quiz_started / box_viewed` | 15-25% | <12% for 2 days |
| Pass Rate | `quiz_passed / quiz_started` | 60-80% | <50% for 2 days |
| Avg Quiz Time | avg(`elapsed_seconds`) valid attempts | 20-60 sec | <12 sec or >90 sec |
| Verified Read Cost | `(net_credits_cost + review_cost) / verified_reads` | within budget | >budget for 2 days |
| Fraud Flag Rate | `fraud_signals / attempts` | manageable | >threshold set in pilot |
| Redemption Rate | `redeem_users / users_with_credits` | monitor trend | sudden spikes >2x baseline |

## Data Revenue KPIs (v0.2)

| KPI | Formula | Target | Alert |
|---|---|---|---|
| Annotation Task Yield | `valid_annotation_labels / annotation_tasks_served` | >80% | <65% for 2 days |
| Dataset Export Quality | `pairs_with_consensus / total_pairs` | >75% | <60% for 2 days |
| Engagement Score Avg | mean(`engagement_score`) active stories | >40/100 | <30 for 3 days |
| Data Revenue per Verified Read | `(annotation_revenue + analytics_revenue) / verified_reads` | monitor positive trend | flat or negative for 7 days |

## KPI update SLA
- Dashboard refresh every 15 min.
- Daily KPI report generated by 19:00.

## Decision logic
- GO: all three gates pass.
- CONDITIONAL_GO: one gate misses but trend improving with clear remediation.
- NO_GO: critical gate fails without realistic remediation within 2 weeks.
